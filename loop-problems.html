Problem 3: Loop Problems

On the past CCC, Alex forgot to account for loops correctly. You like loops, so you decide to make a problem in honor of loops, to show that they arenâ€™t always bad. Given a graph with vertices labelled 1 to N and M edges, determine the length of the longest cycle.

Input Specifications

The first line will contain two space separated integers, N and M.
M lines will follow, each containing two space separated integers 1 ≤ x ≤ N and 1 ≤ y ≤ N, indicating there is an edge from x to y but not vice versa. (x ≠ y and no pair (x,y) will be repeated)

Output Specifications

A single integer, the length of the longest loop in the graph. It is guaranteed that a loop always exists.

Subtasks

For 20% of the 100 marks, N ≤ 100
For an additional 20% of the 100 marks, N ≤ 1000
For all points, N ≤ 100 000

Sample Input

2 2
1 2
2 1

Sample Output

2

Sample Input

3 6
1 2
2 3
3 1
1 3
3 2
2 1

Sample Output

3
